# LEIA PRIMEIRO

Ordem obrigatoria de leitura para qualquer agente.

1. `IMPLANTAR/MAIN-ONLY.md`
2. `IMPLANTAR/CAIXA-DE-ENTRADA.md`
3. `IMPLANTAR/CAIXA-DE-SAIDA.md`
4. `IMPLANTAR/back-end/README.md` se for trabalho de backend
5. pasta da frente correspondente

## Regras curtas
- trabalhar sempre no `main`
- fazer `git pull origin main` antes de iniciar
- fazer commit e push a cada movimentacao relevante
- responder na caixa de saida ao concluir ou bloquear
- nao pegar a mesma tarefa de outro agente

## Estrutura
- regras ficam na raiz de `IMPLANTAR`
- trabalho de backend fica em `IMPLANTAR/back-end`
- trabalho de frontend e PWA pode ficar em pastas irmas dentro de `IMPLANTAR`
