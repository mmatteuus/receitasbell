# Back-end

Esta pasta e a trilha canonica do back-end dentro de `IMPLANTAR`.

## Ordem de leitura para agentes de backend
1. `IMPLANTAR/00-LEIA-PRIMEIRO.md`
2. `IMPLANTAR/MAIN-ONLY.md`
3. `IMPLANTAR/CAIXA-DE-ENTRADA.md`
4. `IMPLANTAR/back-end/STATUS-BACK-END.md`
5. `IMPLANTAR/back-end/TAREFAS-BACK-END.md`
6. `IMPLANTAR/back-end/dossies/*`

## Objetivo
- concentrar tudo que e de back-end aqui
- separar de front-end e PWA
- manter uma trilha unica para auditoria, handoff e execucao

## Escopo desta pasta
- auditoria de rotas
- Stripe backend
- webhooks
- health checks
- tarefas e dossies tecnicos do back-end

## Regras
- agentes de backend devem registrar resposta na caixa de saida
- agentes de backend devem referenciar esta pasta ao documentar novas tarefas
- novas auditorias de backend devem nascer dentro desta pasta
